从这学期开学到现在两个月写了70多道题，最近明显感觉遇到瓶颈了，所以到年底前暂时不打算往后写了。准备把自己做过的题好好总结一下，尽量达到一题多解。于是就有了这个“一题多解LeetCode”系列，作为自己做题总结的电子存档版。

1:两数之和

题目：给定一个整数数组 nums 和一个目标值 target，请你在该数组中找出和为目标值的那 两个 整数，并返回他们的数组下标。

你可以假设每种输入只会对应一个答案。但是，你不能重复利用这个数组中同样的元素。

示例:

给定 nums = [2, 7, 11, 15], target = 9

因为 nums[0] + nums[1] = 2 + 7 = 9
所以返回 [0, 1]

第一题，LeetCode里的“A+B Problem”，这题还是很简单的

最容易想到的方法：暴力法

class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        vector<int> res;
        int len = nums.size();
        for(int i = 0; i < len ;i ++){
            for(int j = 0; j < len; j++){
                if(nums[i] + nums[j] == target && i != j){ 
                    // 因为题目要求数字不能重复使用所以增加判断 i!=j 
                    res.push_back(i);
                    res.push_back(j);
                    return res;
                    //相等就直接返回
                }
            }
        }
        return res;
    }
};

分析：

两重循环所以时间复杂度为O(n2)O(n^2)O(n2)，空间上只增加了一个长度为2的vector可以忽略不计，空间复杂度为O(1)O(1)O(1)

还能更好吗：

暴力法中的两重循环，第一重循环选取起始的数字，看起来不能继续简化，第二重循环寻找加nums[i]后等于target的数字，可以使用map匹配减少时间复杂度。

改进后的方法：一遍哈希表

class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        int len = nums.size();
        map<int,int> hash;
        vector<int> res;
        //先把所有数字和下标存进map中，注意这里因为是数字匹配，
        //所以键值Key为数字nums[i],值value为下标i
        for(int i=0;i<len;i++){
            hash.insert(map<int,int>::value_type(nums[i],i));
        }
        for(int i=0;i<len;i++){
            int temp = target - nums[i];
            //temp为target减去nums[i]的剩余值
            if(hash.find(temp) != hash.end() && hash[temp] != i){
            //同样 hash[temp] != i 保证数字没有重复
                res.push_back(i);
                res.push_back(hash.find(temp)->second);
                break;
            }
        }
        return res;
    }
};
